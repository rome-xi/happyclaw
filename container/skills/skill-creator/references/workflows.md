# Skill 设计模式参考

## 流程型 Skill

适合有明确步骤顺序的场景。

```markdown
## 工作流程

### 1. 获取输入
从用户消息中提取参数。

### 2. 执行操作
调用脚本或 API。

### 3. 返回结果
格式化输出返回给用户。
```

## 任务型 Skill

适合提供多种独立操作的场景。

```markdown
## 快速开始
最常见用法的简要说明。

## 创建文档
具体步骤...

## 编辑文档
具体步骤...

## 导出文档
具体步骤...
```

## 参考型 Skill

适合规范、标准或配置类场景。

```markdown
## 规范

### 命名规则
...

### 格式要求
...

## 用法示例
...
```

## 渐进式加载

Skill 有三层加载机制：

1. **元数据**（name + description）— 始终在上下文中，约 100 词
2. **SKILL.md 正文** — skill 触发后加载，控制在 500 行以内
3. **references/** — Agent 按需读取，不限大小

SKILL.md 超过 500 行时应拆分到 references/。

## 脚本使用原则

- 重复执行的代码放 `scripts/`
- 脚本可直接执行，无需加载到上下文
- 脚本需要测试确保可用
- 保持脚本独立，避免复杂依赖
