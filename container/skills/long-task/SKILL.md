# /long-task — 长期任务执行

支持长时间自主运行的复杂任务，分阶段执行和汇报。

## 触发场景

- 用户提出明显耗时的任务（大规模重构、批量处理、深度研究）
- 用户说"你自己搞"、"后台跑"、"慢慢来"
- 任务预计需要多个步骤且每步都可能耗时较长

## 执行策略

### 1. 任务分解

收到任务后立即拆解为可独立执行的子任务：

```
大任务
├── 阶段 1：调研/准备（了解现状、收集信息）
├── 阶段 2：核心执行（主要工作）
├── 阶段 3：验证/测试（确认结果正确）
└── 阶段 4：收尾/汇报（整理产出、通知用户）
```

使用 TodoWrite 工具跟踪所有子任务状态。

### 2. 分阶段汇报

每完成一个阶段，通过 `send_message` MCP 工具向用户推送进度：

```
阶段 1/4 完成 ✓
- 已完成：[具体内容]
- 发现问题：[如有]
- 下一步：[阶段 2 的计划]
```

### 3. 阻塞处理

遇到阻塞时的策略：

| 情况 | 处理 |
|------|------|
| 需要用户决策 | 发消息说明选项，继续可并行的其他子任务 |
| 技术障碍 | 尝试备选方案，记录失败原因 |
| 依赖外部服务 | 设置重试，同时推进不依赖该服务的任务 |

### 4. 并行执行

独立子任务尽量并行：

- 使用 Task 工具启动子代理处理独立工作
- 设置 `run_in_background: true` 让子任务后台运行
- 主线程继续推进其他工作

### 5. 最终汇报

任务全部完成后发送总结：

```
任务完成 ✓

完成内容：
- [列表]

产出文件：
- [路径列表]

耗时：约 X 分钟
```

## 适用任务类型

| 类型 | 示例 |
|------|------|
| 代码重构 | 大规模重命名、架构迁移、依赖升级 |
| 批量处理 | 文件格式转换、数据清洗、批量下载 |
| 深度研究 | 技术选型调研、竞品分析、文档梳理 |
| 项目初始化 | 搭建项目骨架、配置 CI/CD、编写文档 |
| 数据迁移 | 数据库迁移、配置迁移、日志归档 |

## 注意事项

- 每个阶段完成后都要 commit 中间结果（如涉及代码变更）
- 不要在一个超长步骤中做太多事，保持每步可回滚
- 遇到不确定的决策点，倾向于保守选择，记录下来后续可调整
- 长任务期间如果用户发来新消息，优先响应用户
