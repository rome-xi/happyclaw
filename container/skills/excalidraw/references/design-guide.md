## 流程图 Flowchart

**布局**：TB（上到下）为主，节点居中对齐
**元素**：
- 开始/结束 → 椭圆（灰色 #e9ecef）
- 处理步骤 → 矩形（蓝色 #a5d8ff）
- 条件判断 → 菱形（黄色 #ffec99）
- 输入/输出 → 平行四边形（用矩形 + angle 模拟，或用普通矩形标注）

**间距**：节点间 LAYER_GAP=100px，分支 SIBLING_GAP=60px
**箭头标签**：条件分支用「是/否」或「成功/失败」标注，放在箭头旁边作为独立 text

## 架构图 Architecture

**布局**：分层（从上到下或从左到右），每层用背景区域矩形分组
**元素**：
- 层背景 → 大矩形（#f8f9fa 填充，#dee2e6 描边，roughness=0）
- 层标题 → 独立 text，放在背景左上角（不要用 boundElements）
- 组件 → 矩形，按语义配色
- 连线 → 箭头，使用虚线 strokeStyle="dashed" 表示可选依赖

**推荐分层**：
```
用户层 (Client)     → 蓝色系
网关层 (Gateway)     → 紫色系
服务层 (Services)    → 绿色系
数据层 (Data)        → 中性灰色系
```

**注意**：背景区域放在 elements 数组最前面，组件放在后面

## 思维导图 Mindmap

**布局**：中心节点居中，分支向四周扩散
**元素**：
- 中心节点 → 较大矩形或椭圆（主色）
- 一级分支 → 矩形，每个方向一种颜色
- 二级分支 → 较小矩形，颜色同父级但更浅
- 连线 → 曲线（不用箭头头部）

**配色策略**：每个一级分支分配不同的语义色
- 分支1: 蓝色 #a5d8ff / #1971c2
- 分支2: 绿色 #b2f2bb / #2f9e44
- 分支3: 黄色 #ffec99 / #e67700
- 分支4: 紫色 #d0bfff / #7048e8
- 分支5: 红色 #ffc9c9 / #e03131

**间距**：一级分支间隔 200px+，二级分支间隔 80px

## 时序图 Sequence

**布局**：LR（参与者水平排列，消息从上到下）
**元素**：
- 参与者 → 矩形（顶部和底部各一个，通过虚线连接）
- 生命线 → 竖直虚线（line, strokeStyle="dashed"）
- 消息 → 水平箭头（同步=实线，异步=虚线）
- 返回值 → 水平虚线箭头
- 激活条 → 窄矩形覆盖在生命线上

**间距**：参与者间 200px，消息间 60px
**注意**：消息文本放在箭头上方，作为独立 text

## 类图 Class Diagram

**布局**：TB，使用分栏矩形
**元素**：
- 类 → 用 3 个矩形堆叠（类名/属性/方法），通过 groupIds 分组
  - 类名栏：较深颜色背景
  - 属性栏：白色背景
  - 方法栏：白色背景
- 关系 → 不同箭头样式：
  - 继承：空心三角 (endArrowhead="triangle")
  - 实现：虚线+空心三角
  - 组合：实心菱形 (startArrowhead="diamond") — 用自定义标注
  - 关联：普通箭头

**间距**：类之间 SIBLING_GAP=80px, LAYER_GAP=120px

## ER 图 Entity Relationship

**布局**：自由布局或网格，实体矩形 + 关系菱形
**元素**：
- 实体 → 矩形，标题栏+属性列表（可用分栏矩形）
- 关系 → 菱形或直接用箭头标签
- 基数标注 → 独立 text（1, N, 0..*, 1..*）

**配色**：实体用蓝色系，弱实体用紫色系，关系用黄色菱形

## 状态图 State Diagram

**布局**：TB 或自由布局
**元素**：
- 初始状态 → 小实心圆（椭圆，黑色填充，40x40）
- 普通状态 → 圆角矩形
- 终止状态 → 双圈（两个同心椭圆）
- 转换 → 箭头 + 事件/条件标签

**配色**：当前状态用绿色，错误状态用红色，普通用蓝色

## 组织架构图 Org Chart

**布局**：TB，严格的树形结构
**元素**：
- 节点 → 矩形，包含姓名和职位（两行文本）
- 连线 → 肘形箭头 (elbowed=true)

**配色**：按层级渐变 — CEO 最深色，下级逐渐变浅
**间距**：SIBLING_GAP=60px, LAYER_GAP=80px

## 时间线 Timeline

**布局**：LR，横向展开
**元素**：
- 时间轴 → 水平长线
- 里程碑 → 时间轴上的圆点（小椭圆）
- 事件 → 矩形，交替放在时间轴上方和下方
- 连线 → 从圆点到事件矩形的短线

**配色**：按时间阶段分色，已完成用绿色，进行中用蓝色，未来用灰色

## 看板 Kanban

**布局**：LR，列容器 + 卡片
**元素**：
- 列容器 → 高矩形背景（#f8f9fa）
- 列标题 → 独立 text，居中在列顶部
- 卡片 → 小矩形，白色背景，灰色描边
- 任务数 → 独立 text（标题旁的计数）

**列配色**：
- 待办 → 灰色列头
- 进行中 → 蓝色列头
- 已完成 → 绿色列头

**间距**：列间 40px，卡片间 20px，卡片内边距 16px

## 网络拓扑 Network

**布局**：自由或分层
**元素**：
- 服务器/路由器 → 矩形或椭圆，按类型配色
- 客户端 → 小矩形
- 连线 → 双向箭头或无箭头线（表示双向通信）
- 网段 → 背景区域矩形

**配色**：核心设备用蓝色，边缘设备用绿色，外部用灰色

## 柱状图 Bar Chart

**布局**：固定坐标系
**元素**：
- X 轴 → 水平线
- Y 轴 → 垂直线
- 柱子 → 矩形，从 X 轴向上生长
- 标签 → 独立 text
- 数值 → 柱顶独立 text

**技巧**：所有柱子宽度一致（40-60px），间距一致（20-30px）。颜色按类别分配。

## 饼图 Pie Chart

**布局**：用多个扇形近似（Excalidraw 没有原生扇形）
**替代方案**：
- 用矩形色块 + 百分比文本模拟图例
- 用带标注的圆（椭圆）+ 引线文本
- 或用 freedraw 画近似扇形边界

**推荐**：画一个大椭圆，用 line 从圆心画分割线，配合颜色标注

## 折线图 Line Chart

**布局**：固定坐标系
**元素**：
- 坐标轴 → 两条线（X 和 Y）
- 数据线 → line 元素，多个 points
- 数据点 → 小椭圆放在折点上
- 标签 → 独立 text

## SWOT 分析

**布局**：2×2 网格
**元素**：
- 4 个大矩形，紧密排列：
  - 左上 S (Strengths) → 绿色 #b2f2bb
  - 右上 W (Weaknesses) → 红色 #ffc9c9
  - 左下 O (Opportunities) → 蓝色 #a5d8ff
  - 右下 T (Threats) → 黄色 #ffec99
- 每个象限标题 → 独立 text（大号字）
- 列表项 → 多行文本或多个独立 text

**尺寸**：每象限 300×200px，总尺寸 600×400px

## 决策树 Decision Tree

**布局**：TB，二叉树结构
**元素**：
- 决策节点 → 菱形（黄色）
- 结果节点 → 矩形（绿色=好结果，红色=坏结果）
- 连线 → 箭头 + 条件标签

**间距**：每层深度加倍展开宽度，避免节点重叠

## 线框图 Wireframe

**布局**：设备框内的 UI 组件
**元素**：
- 设备框 → 大矩形，圆角，灰色描边
- 导航栏 → 顶部矩形条
- 按钮 → 小圆角矩形
- 文本框 → 矩形 + 内部虚线
- 图片占位 → 矩形 + X 交叉线
- 文本行 → 灰色水平线（表示占位文本）

**配色**：全灰色系（#495057 描边，#f8f9fa 填充），突出交互元素用蓝色
**roughness**：设为 0（精确线条更适合 UI 线框）
