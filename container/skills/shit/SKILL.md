# /shit — 工作区清理与精简

排泄无用产物，保持工作区健康。

## 触发场景

- 用户要求清理工作区
- 用户说"拉个屎"、"清理一下"、"瘦身"
- 工作区积累了大量临时文件

## 清理范围

### 可自动清理（直接执行）

| 目标 | 路径模式 | 说明 |
|------|---------|------|
| Agent 日志 | `logs/*.log` | 容器运行日志 |
| 临时文件 | `*.tmp`, `*.bak`, `*.swp` | 编辑器/脚本临时文件 |
| Python 缓存 | `__pycache__/`, `*.pyc` | Python 字节码缓存 |
| Node 缓存 | `node_modules/.cache/` | 构建缓存 |
| 系统垃圾 | `.DS_Store`, `Thumbs.db` | OS 元数据 |
| 空目录 | 空的子目录 | 残留的空文件夹 |

### 需汇报后执行（告知用户再清理）

| 目标 | 条件 | 说明 |
|------|------|------|
| 大文件 | >10MB 的非必要文件 | 下载的 PDF/图片/数据文件 |
| 旧下载 | `downloads/` 中超过 30 天的文件 | IM 下载的文件 |
| 过期记忆 | 超过 90 天的日期记忆 | 提炼有用信息后可归档 |

### 永不自动清理（红线）

- `*.md` 文件（记忆、文档）
- `conversations/` 目录（对话归档）
- `CLAUDE.md`（系统记忆）
- `.claude/` 目录（会话配置）
- 任何配置文件

## 执行流程

### 1. 扫描

```bash
# 扫描可清理项目，汇总大小
find /workspace/group -name "*.tmp" -o -name "*.bak" -o -name "*.pyc" -o -name "__pycache__" -o -name ".DS_Store" 2>/dev/null
du -sh /workspace/group/logs/ 2>/dev/null
```

### 2. 安全检查

- 单次命中 >10 个文件或 >200MB 时，降级为 dry-run（只报告不执行）
- 列出将要清理的文件，等用户确认

### 3. 执行清理

可自动清理的项目直接删除，需确认的项目汇报后等指示。

### 4. 汇报

格式：
```
清理完成：
- 删除了 X 个临时文件（Y MB）
- 清理了 logs/ 目录（Z MB）
- 发现 N 个大文件待确认：[列表]
- 工作区当前占用：总计 W MB
```

## 安全规则

- 删除前先移到 `.trash/` 目录（如目录不存在则创建）
- `.trash/` 本身可在下次清理时删除（确认上次清理无误后）
- 超过安全阈值时只报告不执行
- 永远不碰红线文件
